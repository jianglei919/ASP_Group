# 需求总结

## 来源
- 基于文档：Project_W26.pdf
- 课程：COMP-8567
- 截止日期：2026/04/13

## 1. 项目目标
构建一个基于 Socket 的客户端-服务器系统，使客户端能够在服务端以用户主目录（~）为根的目录树中查询并获取文件。
系统包含三个服务程序：
- w26server（主服务）
- mirror1（服务副本）
- mirror2（服务副本）

客户端发送项目定义命令（非 Linux 原生命令）来请求：
- 目录列表
- 单文件元信息
- 按大小/类型/日期筛选后打包的压缩文件（temp.tar.gz）

## 2. 系统范围与约束
- 通信方式必须仅使用 Socket。
- w26server、mirror1、mirror2 与客户端需运行在不同机器或不同终端。
- 服务端检索范围限定为其主目录（~）下的目录树。
- 服务端返回的文件必须保存在客户端主目录下的 project 文件夹（~/project）。

## 3. 服务端需求（w26server / mirrors）
### 3.1 启动与监听
- w26server、mirror1、mirror2 必须在任意客户端启动前先启动。
- 各服务端启动后持续等待客户端请求。

### 3.2 单客户端进程模型
- 每当有客户端连接时，服务端必须 fork 一个子进程。
- 子进程在 crequest() 中独占处理该客户端请求。
- 父进程返回监听状态，继续接收其他客户端连接。
- crequest() 需在无限循环中接收并处理命令。
- 收到 quitc 后，crequest() 退出。

### 3.3 请求处理行为
对每个合法命令，服务端必须：
- 解析命令参数
- 执行对应的文件系统检索/筛选
- 返回文本结果或 temp.tar.gz
- 无匹配时返回规定的未找到提示

## 4. 客户端需求
### 4.1 交互循环
- 客户端在无限循环中等待用户输入命令。

### 4.2 命令校验
- 客户端必须在发送前完成命令语法校验。
- 若语法错误，客户端输出相应错误提示。
- 仅将语法正确的命令发送给服务端。

### 4.3 终止行为
- 输入 quitc 时，客户端发送该命令并终止进程。

## 5. 按命令划分的功能需求
### 5.1 dirlist -a
- 服务端返回其主目录下所有子目录（仅文件夹）。
- 排序方式：按字母升序。
- 客户端打印返回列表。

### 5.2 dirlist -t
- 服务端返回其主目录下所有子目录（仅文件夹）。
- 排序方式：按创建时间升序（最早创建在前）。
- 客户端打印返回列表。

### 5.3 fn filename
- 服务端在以 ~ 为根的目录树中查找 filename。
- 找到时返回：
  - 文件名
  - 文件大小（字节）
  - 创建日期
  - 文件权限
- 若存在多个同名文件，仅返回首次成功匹配结果。
- 若未找到，客户端输出：File not found。

### 5.4 fz size1 size2
- 返回 temp.tar.gz，包含满足以下条件的文件：
  - size >= size1
  - size <= size2
- 参数约束：size1 <= size2，且 size1 >= 0、size2 >= 0。
- 若无匹配文件，返回：No file found。

### 5.5 ft <extension list>
- 扩展名列表至少 1 个、最多 3 个，且应为不同文件类型。
- 返回 temp.tar.gz，包含匹配扩展名的所有文件。
- 若无匹配文件，返回：No file found。

### 5.6 fdb date
- 返回 temp.tar.gz，包含创建日期 <= date 的文件。
- 示例日期格式：YYYY-MM-DD。

### 5.7 fda date
- 返回 temp.tar.gz，包含创建日期 >= date 的文件。
- 示例日期格式：YYYY-MM-DD。

### 5.8 quitc
- 命令发送到服务端后，客户端进程终止。

## 6. 连接分流需求（主服务与镜像轮转）
连接处理顺序必须满足：
1. 第 1-2 个连接由 w26server 处理
2. 第 3-4 个连接由 mirror1 处理
3. 第 5-6 个连接由 mirror2 处理
4. 从第 7 个连接起按循环轮转：
   - 7 -> w26server
   - 8 -> mirror1
   - 9 -> mirror2
   - 后续持续重复

## 7. 错误处理需求
- 客户端命令语法错误：客户端输出错误提示，不进入服务端处理流程。
- fn 无匹配：返回/输出 File not found。
- fz、ft、fdb、fda 无匹配：返回/输出 No file found。

## 9. 验收检查清单（实践）
- 三个服务端在客户端启动前已就绪并处于监听状态。
- 每个客户端连接均由 fork 的子进程独立处理。
- 所有命令均可运行，且客户端语法校验有效。
- w26server、mirror1、mirror2 的连接分流顺序正确。
- 返回压缩包为有效 temp.tar.gz，并保存在客户端 ~/project。
- 各类未找到提示与题目规定完全一致。
